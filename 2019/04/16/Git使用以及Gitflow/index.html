Git使用以及Gitflow

发表于 2019-04-16 | 分类于 Git | | 阅读次数:

  Git是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理，接下来我们会循序渐进地介绍Git的主要命令。该篇博客的主要内容以及图片素材来自于https://learngitbranching.js.org/。

Git Commit

  Git仓库中的提交记录保存的是你的目录下的所有文件的快照，就像是把整个目录复制然后再粘贴一样，但比复制粘贴更优雅！Git希望提交记录尽可能的轻量，因此每次你在提交时，它不会忙盲地复制整个目录。在条件允许的情况下，它会将当前版本与上一个版本进行对比，并把所有的差异打包到一起作为一个提交记录。Git还保留了这种关系，这也是为什么大多数提交记录的上面都有父节点的原因，对于项目组的成员来说，维护提交历史对大家都有好处。
如图有两个提交记录c0、c1，我们修改代码库并将修改保存为一个提交记录c2，父节点是当前提交中的变更基础。

Git Branch

  Git的分支也非常的轻量。他们只是简单的指向某个提交记录 – 仅此而已，所以许多的Git爱好者传颂：早建分支！多用分支！ 这是因为即使创建再多的分支也不会造成存储或者是内存上的开销，而且按照逻辑分解工作到不同的分支要比维护那些臃肿的分支简单多。在将分支以及提交记录结合之后，我们会看到两者如何协作。现在只要记住分支其实就是相当于“我想基于这个提交以及它所有的父提交进行新的工作。”
